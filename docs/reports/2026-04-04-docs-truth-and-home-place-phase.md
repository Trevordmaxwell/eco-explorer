# Docs Truth And Home-Place Phase

## Summary

The latest external review is directionally positive, but it calls out two next-step needs that belong together in lane 1:

- public docs and tooling notes should keep matching the real game and exposed commands
- the field station and nursery should show player progress more visibly, not just track it

This wave should first keep the repo's public story truthful, then deepen the visible home-place payoff without adding another shell, dashboard, or copy-heavy layer.

## Goals

- keep `README`, architecture notes, authoring docs, and tooling references aligned with the live game
- clean up naming seams where older nursery or chapter labels no longer match the current route-facing language
- make the station feel more shaped by the player's season through visible change rather than more text

## Wave Shape

### Step 1

- prepare a docs, tooling, and naming sync audit

### Step 2

- implement the public docs and naming sync pass

### Step 3

- review whether the public repo story now matches the live build and exposed commands

### Step 4

- prepare a stronger visible home-place transformation handoff

### Step 5

- implement one more visibly transformative station or nursery payoff using existing seams

### Step 6

- review whether the home place now feels more changed by the player's learning

## Guardrails

- do not add a new station page or planner shell
- do not inflate the copy budget to solve payoff problems
- prefer visible state, motion, and small environmental payoffs over more words
- keep docs truthful to the local repo rather than stale public assumptions
