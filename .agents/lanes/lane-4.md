# Lane 4: Retired Into Lane 1

Lane 4 is not an active lane in the current director-plus-two-lanes model.

Former lane 4 responsibilities now route through:

- `lane-1`: playability, systems, station, routes, support, replay, notebook filing, and integration proof

Historical reports, packets, and done queue items may still mention lane 4. Treat those labels as archive context unless the director explicitly reactivates lane 4.

If a fresh agent is assigned lane 4, it should stop and ask whether the work should be routed to lane 1.

## Legacy Scope

The old lane 4 scope was:

- route semantics and Route v2 conversion work
- evidence-backed outing beats
- tiny pre-outing support choice
- station-side notebook synthesis and route completion
- soft replay windows tied to existing world-state

Carry that spirit forward under lane 1.
